# 预测市场平台接入

> 目标：聚合多个预测市场的世界杯事件，知道"有哪些可下注的预测事件"+ 跨平台赔率对比
> 优先级：Polymarket（第一）> Betfair/Pinnacle（传统盘口对照）> 其他预测市场

---

## 一、为什么要接预测市场（而非只看博彩赔率）

| 维度 | 传统博彩（Pinnacle等） | 预测市场（Polymarket） |
|------|---------------------|----------------------|
| 定价机制 | 庄家设盘 + 抽水 | 用户买卖，价格=隐含概率 |
| 抽水 | 4-7% | 极低（链上手续费） |
| 信息 | 庄家信息领先 | 群体智慧，有时滞后 |
| 套利 | 平台间限制多 | 与博彩盘存在跨市场价差 |
| 事件颗粒 | 固定盘口 | 可有长尾事件（夺冠/小组头名/特定剧情） |

**核心机会**：Polymarket 价格（隐含概率）与传统博彩赔率（隐含概率）之间出现背离时 → 跨市场套利或择优下注信号。

## 二、Polymarket API 架构（已确认）

Polymarket 把功能拆成三套 API：

### 1. Gamma API — 市场发现 / 元数据（无需认证）
- 用途：**列出有哪些预测事件**、市场标题、结束时间、分类、token id
- 这是"知道有哪些预测事件"的核心入口
- 基础域名：`https://gamma-api.polymarket.com`
- 关键能力：按标签/分类筛选（如 `sports`、`soccer`、`world-cup`），返回 events 与其下属 markets

### 2. CLOB API — 中央限价订单簿 / 价格 / 交易
- 基础域名：`https://clob.polymarket.com`
- 公开端点（无需认证）：
  - `GET /price` — 某 token 当前价格（= 隐含概率）
  - `GET /book` — 订单簿深度（判断流动性，决定能下多大仓）
  - `GET /midpoint` — 中间价
- 认证端点（HMAC-SHA256 签名 + API Key）：
  - `POST /order` — 下单
  - `DELETE /order` — 撤单

### 3. Data API — 用户持仓 / 成交历史
- 用途：跟踪自己的持仓、盈亏、历史成交（对接复盘库）

### ✅ 已实测验证的拉取流程（2026-06-10）

**世界杯专题 = Gamma API tag_id `102350`（"2026 FIFA World Cup"）**
（相关 tag：`102232` FIFA World Cup 通用、`100350` Soccer、`1` Sports）

```bash
# 拉专题下所有活跃事件，按成交量排序
curl -s "https://gamma-api.polymarket.com/events?tag_id=102350&active=true&closed=false&order=volume&ascending=false&limit=200"
```
- 返回字段：每个 event 含 `title/slug/volume/liquidity/markets[]/tags[]`
- 每个 market 含 `groupItemTitle`（如队名）、`outcomePrices`（字符串化数组，[0]=Yes概率）、`volume`、`outcomes`、token id
- 分页：`limit`(≤500) + `offset`，响应看 `has_more`
- 找专题 tag 的方法：随便取一个已知 event（如 slug=world-cup-winner），读它的 `tags[]` 字段即可反查

### 实测数据快照（2026-06-10）
- 专题总成交 **$18.73 亿**，"World Cup Winner" 独占 **99.3%**（$18.6亿）
- 夺冠概率 Top：西班牙16.2% / 法国16.1% / 英格兰10.8% / 葡萄牙10.2% / 阿根廷8.6% / 巴西8.5%
- 共约 100 个事件：夺冠、金靴、出线、各轮晋级(R16/QF/SF/Final)、各队淘汰阶段、小组垫底/次名、球员盘、花边盘

### ⚠️ 关于低流动性的正确理解（展示原则·铁律）
**低流动 ≠ 剔除**。大量事件流动性低，往往只是**刚上线、还没人关注**，不代表没意义。
铁律：**低流动/低关注绝不作为剔除展示的理由**，平台照常全面呈现、不漏盘。

> ⚠️ 注意边界：这是**展示原则**（不漏盘），**不是下注edge**。
> 原"先发布局(策略N)/抢在流动性进来前拿最优价"已**删除**（属抢跑信息差，[见铁律](../3_策略分析库/核心策略清单.md)）。
> 我们不靠"比市场更早"获利，只保证盘面呈现完整。

**操作含义（仅展示层）**：
- 持续监控新上线事件（每日扫 tag 102350，对比昨日 diff，标出**新增市场**用于呈现）
- `is_new` 仅作**展示标记**，不附带"该下注"含义
- 不因"成交量小"剔除任何盘——覆盖度优先

## 八、数据源完整性 & 稳定性核查（2026-06-10 完成）

### 数据源地图（两条线，缺一不可）
| 数据 | 入口 | 内容 | 数量 |
|------|------|------|------|
| **赛事长盘** | tag `102350` | 夺冠/金靴/出线/各轮晋级/各队淘汰阶段/球员盘/花边盘 | 149活跃 + 19已结束 |
| **单场比赛盘** | series `11433`（= tag `102232`） | 每场比赛的所有玩法 | **348** 个场次事件 |

> 找专题/系列 id 的方法：取任一已知 event 读其 `tags[]`；或 `/sports` 端点按 `sport` 反查 `series`。
> `fifwc`(单场) → series 11433 / tag 102232；专题聚合 → tag 102350。

### ✅ 单场盘玩法完整（所有策略均可执行）
每场比赛含：
- **1X2** 主胜/平/客胜 → 策略 A/B/C/G
- **O/U 总进球 0.5–5.5** + 半场 O/U → **策略 A 强打弱大分** ✓
- **准确比分 Exact Score**（含 0-0…3-3 + Any Other）→ **策略 F** ✓
- **球队让球 (-1.5) + 单队 O/U** → 策略 B 亚盘 ✓
- **Both Teams to Score / 半场 BTTS** → 衍生盘
- **Total Corners 角球** → 策略 F 辅助盘
- **Player Props 球员盘**

字段：`startTime`/`endTime` = 真实开赛时间(UTC)；`outcomePrices[0]` = 该选项隐含概率。
开幕战已挂出：Mexico vs South Africa，2026-06-11 19:00 UTC，墨西哥胜 0.69 / 平 0.20 / 南非 0.10。

### ✅ 抽签已反映
单场盘里小组对阵真实存在（如 France vs Senegal、Spain vs Cabo Verde…）→ 可从 series 11433 反推完整分组，无需另找抽签源。

### ✅ 稳定性
- 连续 5 次请求：5/5 HTTP 200，字节数完全一致(6551375)，延迟 1.1–1.8s
- 读端点全部**无需认证**，无 API key
- ⚠️ 单页硬上限 **100 条**（`limit>100` 无效，必须 offset 翻页）
- ⚠️ urllib 直连被 403（缺 User-Agent）→ 脚本里务必带 UA，或用 curl

### ⚠️ Polymarket 数据覆盖不到的（需外部源补充）
| 缺口 | 补充源 |
|------|--------|
| 实时比分/赛况 | Polymarket Sports WS `wss://sports-api.polymarket.com/ws`；或 API-Football |

### 核查结论
**数据源完整、稳定、可用。** 两条线(tag 102350 长盘 + series 11433 单场)即可覆盖全部已定义策略的下注标的；

## 三、接入分阶段路线

### 阶段1：只读 · 事件发现（不下注，先建数据流）
目标：每天拉取 Polymarket 上所有世界杯相关事件，存入本地。
```
1. 调 Gamma API 拉 events，关键词过滤 "World Cup / 队名 / 小组"
2. 对每个 market 调 CLOB /price + /book，记录隐含概率和流动性
3. 存成结构化表：事件名 | 选项 | 当前价格(概率) | 流动性 | 抓取时间
```
风险：低（纯读取，无需钱包）

### 阶段2：跨市场对比
```
1. 同时拉传统博彩赔率（Pinnacle等）转成隐含概率
2. 与 Polymarket 概率对齐同一事件
3. 计算价差，价差 > 阈值（如 5%）→ 标记为机会，推入预测平台"最大盈亏比"榜单
```

### 阶段3：自动/半自动下单
```
1. 用 py-clob-client + Polygon 钱包 + USDC
2. 仓位由用户自定（仓位管理系统已取消，不设自动评分门槛）
3. 先纸面交易（paper trading）跑通逻辑再上真金
```
风险：高（涉及私钥、链上交易、不可逆）。务必先纸面验证。

## 五、世界杯典型预测事件类型（Polymarket 上常见）

| 事件类型 | 例子 | 对应我们的策略 |
|---------|------|--------------|
| 夺冠 | "谁会赢得2026世界杯" | 长期价值，强队确定性(B) |
| 小组头名 | "X队会拿小组第一吗" | 摇摆队博弈(C) |
| 晋级淘汰赛 | "X队会出线吗" | 强队确定性(B)、摇摆队(C) |
| 单场胜负 | "X队会击败Y队吗" | 强打弱(A)、东道主(G) |
| 剧情类 | "会有点球大战吗""会出现帽子戏法吗" | 长尾高赔（高风险·长赔） |

**注意**：Polymarket 多为二元（Yes/No）市场，价格即概率。0.65 = 市场认为65%概率发生。

## 七、策略 M：跨市场价差套利 🚫 不展示（概念存档）

> ⚠️ **用户决定不展示**（2026-06-11）。需接入外部博彩赔率、属博市场无效的打法，
> 不在平台呈现。以下仅作概念存档，不接入、不开发。

**概念**：同一事件，Polymarket 隐含概率 vs 博彩隐含概率价差 ≥ 5% 时，理论上有套利/择优空间。
**为何不做**：① 需接外部博彩数据（工程量+合规）；② 与"平台只做分类呈现、不博市场无效"定位不符。
