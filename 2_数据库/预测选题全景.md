## 三、平台选题数据模型（建议）

平台内部把选题归一到统一结构，便于检索/展示/策略匹配：

```
Topic(选题)
├── id              # Polymarket event/market id
├── line            # "长盘" | "单场"
├── category        # 上面的分类编号
├── match_id        # 单场盘归一到场次（去重用）
├── teams[]         # 涉及队伍
├── bet_type        # 1X2 / O/U / 比分 / 让球 / 出线 / 夺冠 ...
├── strategy_tags[] # 命中的策略 [A,B,C...]
├── price           # 当前隐含概率
├── volume          # 成交量
├── liquidity       # 流动性/订单簿深度
├── is_new          # 是否近期新上线（先发机会）
├── kickoff         # 开赛时间
└── snapshot_ts     # 抓取时间
```
