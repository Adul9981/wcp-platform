## 链接生成规则

### 选题（event）跳转
```
https://polymarket.com/event/{slug}?via=serene77mc-g6kj
```
- `{slug}` 直接取自 Gamma API 的 `slug` 字段（我们已抓取）
- `?via=serene77mc-g6kj` 固定附加，承载推荐归因
- **URL fragment `#xxxx` 可省略**：fragment 不发往服务端，归因不依赖它

### 样本验证（用户提供 2026-06-10）
| 原始链接 | slug | 推断模板一致 |
|---------|------|------|
| highest-temperature-in-shanghai-on-june-10-2026?r=adul#Bb9qQlW | highest-temperature-... | ✅ |
| us-x-iran-permanent-peace-deal-by?r=adul#Bb9qQlW | us-x-iran-... | ✅ |
| world-cup-england-stage-of-elimination?r=adul#iPWlBeR | world-cup-england-stage-of-elimination | ✅ |

→ 规律通用于所有预测市场页面，世界杯选题同样适用。

## 实现约定
1. 平台所有"去 Polymarket 下注"按钮，统一调用一个 `build_event_url(slug)` 函数生成链接。
2. 该函数从配置读取 `POLYMARKET_REFERRAL_CODE`，禁止在多处硬编码 `adul`。
3. 跳转处 UI 透明标注"该链接包含推荐关系"（P6.3 合规）。
4. 若将来推荐码变更，只改此处 / 环境变量，全站生效。
