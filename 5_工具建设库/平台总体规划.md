## 二、技术栈（已定）

| 层 | 选型 | 理由 |
|----|------|------|
| 抓取/分析 | **Python** | 已验证可用，数据生态最强 |
| 后端 API | **FastAPI** | 轻量、异步、自带文档，平滑支撑未来多用户 |
| 存储 | **SQLite → PostgreSQL** | MVP 用 SQLite 零运维；多用户阶段迁 Postgres |
| 定时任务 | **APScheduler / cron** | 每日快照、diff、三榜生成 |
| 前端 | **Next.js (React)** | 公开产品需要真前端；SEO 友好利于获客 |
| 部署 | 单机 Docker → 云 | MVP 单机，后续容器化上云 |

> MVP 阶段前端可先用极简页面甚至先不做，重点是数据管道 + API 跑通。

## 四、核心数据流（MVP 重点）

```
1. 抓取   tag 102350(长盘) + series 11433(单场)  [带UA/翻页/重试]
2. 归一   单场 348 event → 104 场次；统一成 Topic 模型
3. 打标   每个 Topic 自动匹配 strategy_tags [A..N]
4. 快照   存当前 price/volume/liquidity + snapshot_ts
5. diff   对比昨日 → 标 is_new(先发机会)
6. 估值   公平价值（MVP: 跨市场对比/简单规则；后期模型）
7. 三榜   按评分生成 最值得关注/最被忽视/最大盈亏比
8. 输出   写库 → API → 前端 / 文案系统
```

## 五、三榜定义（产品第一屏，恒在）

> ⚠️ **已改版（2026-06-11）**：旧三榜（最值得关注/最被忽视/最大盈亏比）作废。
> 新三榜按**风险**切分，权威定义见 [三榜规则与展示规范](../3_策略分析库/三榜规则与展示规范.md)。

| 榜 | 入选逻辑 | 说明 |
|----|---------|------|
| 📅 今日关注 | 今天开赛/结算的比赛（主区）+ 里程碑/催化剂（副区） | 时间轴，与风险正交 |
| 🛡️ 低风险事件 | 主张 ≤ 队伍"地板"（实力×分组×阶段） | 稳，盈亏比通常低 |
| 🎲 高风险事件 | 长赔子区（≥天花板）+ 摇摆子区（势均力敌） | 博，标盈亏比 |

**门槛**：只靠质量门槛（须有实力/分组/阶段依据），不设数量上限，不按价格砍区间，**低流动不剔除**。
每个条目必含：选题、方向、**盈亏比(必含)**、当前概率、风险档、命中策略、入榜理由、**带邀请码的跳转链接**、数据时间。
> ❌ 不含"公平估值/edge"——未接外部赔率，不声称比市场更准（[不追信息差](../3_策略分析库/核心策略清单.md)）。

## 六、建设里程碑

| 里程碑 | 内容 | 依赖 |
|--------|------|------|
| **M1 数据管道**（MVP核心） | 抓取→归一→打标→快照→diff，稳定落库 | 无 |
| **M2 三榜引擎** | 估值+评分+三榜生成，API 暴露 | M1 |
| **M3 前端展示** | 第一屏三榜 + 选题浏览 + 详情跳转(邀请码) | M2 |
| **M4 文案+复盘自动化** | 赛前/赛后/盘中文案，复盘记录 | M2 |
| **M5 订阅付费** | 账号、付费墙、权限 | M3 |
| **M6 交易(可选/后)** | 纸面→真实，强风控 | 独立 |
