> 目标：每日自动扫描所有赛事，输出结构化预测报告

## 一、平台核心功能

### F1：每日赛事扫描报告

**输入**：
- 当日赛事列表（队伍、时间、小组）
- 两队当前积分/晋级情况
- 最新赔率数据（各大平台）
- 球队伤病/停赛名单

**输出**：三个榜单

**榜单1 — 最值得关注（高确信度机会）**
```
评分依据：策略匹配度 × 赔率价值 × 执行确定性
展示：场次 | 推荐方向 | 当前赔率 | 理论公平赔率 | 优势% | 推荐仓位倍数
```

**榜单2 — 最被忽视（冷门/信息差机会）**
```
评分依据：关注度低 × 有具体信息差 × 赔率未移动
展示：场次 | 忽视原因 | 信息点 | 赔率 | 建议操作
```

**榜单3 — 最大盈亏比（高赔率博弈）**
```
评分依据：赔率 > 5.0 × 有逻辑支撑（非随机）
展示：场次 | 方向 | 赔率 | 逻辑 | 建议仓位（小）
```

---

### F2：实时事件监控（盘中）

**监控事件**：
- 进球（时间、队伍）
- 红牌（时间、位置）
- 点球（判罚/扑救/进）
- 伤停补时公布（分钟数）

**输出**：
- 事件发生后 < 1分钟：推送盘中机会提醒
- 格式：事件 → 赔率变化方向 → 推荐操作 → 窗口时间

---

### F3：策略触发检测

每场赛前，自动检查每条策略的触发条件是否满足：

```
场次：法国 vs [鱼腩]
✅ 策略A：强打弱大分 → 触发（法国S级，对手D级，大分线3.5）
✅ 策略F：准确比分 → 触发（建议3-0、4-0）
❌ 策略D：无需盘中监控（确定性高，无需等待事件）
```

---

### F4：每日复盘自动化

**赛后输入**：比赛结果、关键事件时间轴
**自动生成**：
- 预测方向 vs 实际结果对比
- 命中/未命中分析
- 当日盈亏统计
- 推文草稿（基于复盘模板）

## 二、数据源规划

| 数据类型 | 来源 | 更新频率 |
|---------|------|---------|
| 赔率数据 | OddsPortal / Betfair / Pinnacle API | 实时 |
| 赛事结果 | FIFA官网 / API-Football | 实时 |
| 伤病名单 | Transfermarkt / 官方队伍Twitter | 赛前24h |
| 历史数据 | SoccerSTATS / Statsbomb | 赛前分析 |
| 赛程/积分 | FIFA官网 | 每日 |

## 三、技术实现路径

### 阶段1：MVP（手动+模板）
- 手动填写每日赛事数据
- 使用策略清单人工评分
- 用模板生成推文
- 时间成本：每日 1-2 小时

### 阶段2：半自动化
- Python脚本抓取赔率数据
- 自动计算策略触发条件
- 生成每日报告草稿（人工审核后发布）
- 时间成本：每日 30-45 分钟

### 阶段3：全自动化
- 接入实时赔率API
- 盘中事件WebSocket监控
- AI文案生成（基于模板和数据）
- Telegram机器人自动推送
- 时间成本：每日 15 分钟监控确认

## 四、每日工作流

```
[赛前 D-1]
□ 查看次日赛程
□ 检查双队积分/晋级情况
□ 收集最新伤病名单
□ 查看初始赔率

[赛前 4h]
□ 运行策略触发检测
□ 生成机会评分
□ 编写赛前预测推文
□ 确认下注仓位

[比赛中]
□ 盯盘（或开启事件监控）
□ 执行盘中策略（如触发）

[赛后 1h]
□ 记录结果至复盘库
□ 更新账户余额
□ 生成复盘推文
□ 发布内容

[每周]
□ 汇总周盈亏
□ 策略有效性回顾
□ 调整仓位规则
□ 发布深度分析帖
```

## 五、KPI 指标

| 指标 | 目标 |
|------|------|
| 预测命中率（主方向） | > 52%（超过随机） |
| 期望值（每注平均） | > +5% |
| 月度总收益率 | > +15% |
| 盘中策略命中率 | > 55% |
| 内容互动率 | > 3% |
| 单日最大亏损 | < 5% 总资金 |
