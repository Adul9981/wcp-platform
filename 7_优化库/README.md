## 已记录的优化点

### 📊 数据
- [x] 单场盘按场次去重（348→75，base标题+日期为键，已验证多卡归一）
- [ ] 公平价值估算精度（初期可用跨市场对比，后期上模型）
- [ ] **策略打标精度**：现为标题关键词规则（MVP）。已知粗糙处：
  - "Halftime Result" 被当作完整 1X2 打了 A/B/C/G，应区分半场盘
  - 单场"综合(More Markets)"未拆出具体 O/U 线/让球档，影响策略A精确匹配
  - 改进方向：进入 market 层级按 groupItemTitle 细分打标，而非只看 event 标题
- [ ] 长盘 category 分类器同为关键词规则，边界 case 可能误分到"99其他"(当前7个)

### 🚀 性能
- [ ] 用批量端点 `/books` `/prices`（最多500/次）替代逐个请求
- [ ] 快照增量存储，避免每日全量重存

- [ ] **最被忽视榜区分度不足**：大量0成交新盘并列同分，靠类型限量呈现多样性。
  根因：缺公平价值参照做 tie-break。接外部赔率后用"价差"区分真正错价者。
  ⚠️ 铁律：**低流动性绝不作为剔除理由**（用户2026-06-10明确）——新盘低关注是先发机会的正常态。
- [ ] 公平价值/edge：接 Pinnacle/OddsPortal 后补，区分"真错价"vs"仅低关注"
- [ ] 评分权重需用复盘库实际命中率回测校准（当前为先验权重）

### 🔒 安全
